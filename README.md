## Introduction

MicroEven is an R Shiny web application designed to calculate and visualize microbiome evenness metrics from alpha-diversity outputs generated by MicrobiomeAnalyst. The application addresses a practical gap in current microbiome analysis workflows: although MicrobiomeAnalyst exports richness and Shannon diversity, it does not directly provide alpha-diversity evenness metrics such as Pielou’s evenness. MicroEven fills that gap by allowing researchers to upload MicrobiomeAnalyst output files and compute evenness measures through an accessible graphical user interface rather than custom R scripting. [file:1]

The project is intended for microbiome researchers, students, and other users who may not have extensive programming experience but still need robust, reproducible ecological diversity summaries and visualizations for research reports, theses, posters, and manuscripts. The app was developed at Colorado State University under the supervision of Dr. Tiffany Weir. [file:1]

## Problem Statement

Microbiome researchers commonly use MicrobiomeAnalyst to process alpha-diversity data because it offers a convenient graphical interface for standard metrics such as observed richness and Shannon diversity. However, the platform does not currently offer a graphical workflow for calculating evenness indices, even though evenness is an important ecological concept that helps distinguish communities dominated by a few taxa from communities with more balanced abundance distributions. [file:1]

As a result, users who need evenness measures often have to write custom R scripts or manually process exported data. This creates a barrier for students, non-programmers, and researchers who want a streamlined and reproducible workflow. MicroEven was proposed as a solution that keeps the analysis accessible by providing upload, calculation, visualization, and export capabilities in a browser-based Shiny interface. [file:1]

## Scope

### In Scope

MicroEven is designed to handle MicrobiomeAnalyst alpha-diversity result files in CSV format that contain at minimum `sample_id`, `variable`, and `value` columns, with optional metadata columns such as treatment group, week, visit, or subject identifier preserved for downstream grouping and analysis. The app parses separate Observed Features and Shannon files, validates the expected metric labels, joins the files by sample ID, and computes per-sample evenness metrics. [file:1]

The current implementation computes Pielou’s evenness exactly as \(J' = H'/\ln(S)\) and also calculates a Shannon-derived equitability metric as \(\exp(H')/S\). It then provides per-sample tables, group-level summary tables, publication-style plots, statistical testing options, residual diagnostics for supported models, and export functions for processed data and figures. [file:1]

### Out of Scope

The project does not replace full microbiome analysis pipelines such as quality control, denoising, taxonomic classification, phylogenetic processing, or beta-diversity workflows. It is also not intended to provide production-scale deployment features such as authentication, user accounts, enterprise-grade hosting, or server hardening. Advanced statistical modeling beyond the implemented options, such as mixed models with complex covariance structures or highly customized multivariable analyses, is also outside the present scope. [file:1]

## Core Features

### 1. Input Validation and Import

MicroEven requires two MicrobiomeAnalyst alpha-diversity export files: one for Observed Features and one for Shannon diversity. The parser checks for the required columns `sample_id`, `variable`, and `value`, verifies that the uploaded metric matches the expected file type, removes extraneous row-number columns if present, and preserves metadata columns for later grouping, faceting, and repeated-measures analysis. [file:1]

### 2. Evenness Metric Calculation

After joining the uploaded files by `sample_id`, the app computes two main evenness-related metrics. Pielou’s evenness is calculated exactly from Shannon diversity and observed richness, while Shannon-derived equitability is calculated as a bounded approximation based on the effective number of species from Shannon diversity. The application explicitly notes that this second metric is not true Simpson evenness unless a Simpson index is available. [file:1]

### 3. Tables and Summaries

The app provides an interactive per-sample table showing imported and calculated metrics and a group summary table reporting counts, means, medians, standard deviations, and interquartile ranges across the selected grouping variable. These tables are rendered with searchable and sortable DataTables for usability. [file:1]

### 4. Plotting Module

MicroEven supports multiple plot types, including boxplots, violin plots, and scatterplots against observed richness. Users can choose the metric to visualize, define grouping variables for the x-axis and color mapping, and optionally facet plots using an additional metadata column. Plots are generated with `ggplot2` and formatted for clean presentation. [file:1]

### 5. Statistical Testing

The app includes a statistics module that selects or runs appropriate tests depending on the study design. If a subject ID is provided, the app can fit a linear mixed model with a random intercept per subject for repeated measures; otherwise it supports Wilcoxon rank-sum tests, Kruskal-Wallis tests with Dunn post-hoc comparisons, and parametric t-test or ANOVA alternatives. The implementation also includes residual diagnostics such as Shapiro-Wilk testing and QQ plots where applicable. [file:1]

### 6. Export Options

Users can export per-sample results, group summaries, and statistical results as CSV files. The plotting module also supports figure export as high-resolution PNG and PDF files suitable for manuscripts, presentations, and reports. [file:1]

## Input Requirements

MicroEven expects MicrobiomeAnalyst alpha-diversity files in long format with one row per sample-metric combination. The minimum required columns are:

- `sample_id`: unique sample identifier used for joining files.
- `variable`: metric name, expected to include `Observed` or `Shannon` depending on the uploaded file.
- `value`: numeric value of the metric.

Any additional columns, such as treatment group, week, visit, cage, or subject ID, are preserved as metadata and can be used for grouping, faceting, or repeated-measures analysis in the interface. [file:1]

## Statistical Design Logic

MicroEven was built with microbiome study designs in mind, especially repeated-measures experiments where the same subject may be sampled across time points. In these cases, ordinary independent-sample tests can produce misleading p-values because they ignore within-subject correlation. The app therefore allows the user to select a subject identifier and fit a linear mixed model using `lmerTest`, with pairwise contrasts generated through `emmeans`. [file:1]

When no repeated-measures structure is specified, the application defaults to safer non-parametric options for many small-sample microbiome studies. It automatically chooses Wilcoxon testing for two-group comparisons and Kruskal-Wallis testing for comparisons involving more than two groups, while still allowing a user to manually select parametric alternatives when assumptions are met. [file:1]

## Workflow

The intended user workflow is straightforward:

1. Export Observed Features and Shannon alpha-diversity results from MicrobiomeAnalyst.
2. Upload both files into the MicroEven Shiny interface.
3. Select grouping, faceting, and optional subject ID variables from preserved metadata columns.
4. Compute and inspect per-sample and grouped evenness metrics.
5. Generate boxplots, violin plots, or scatterplots.
6. Run the appropriate statistical test.
7. Export processed tables and plots for manuscripts or presentations. [file:1]

## Recommendations

MicroEven should continue to be developed as a Shiny-based tool with an emphasis on simplicity, robustness, and reproducibility. The core design principle should remain a streamlined workflow built around upload, compute, visualize, and export. Additional effort should focus on strong input validation, clear labeling of metric definitions, and figure outputs that are suitable for publication without extensive post-processing. [file:1]

The project should also preserve a clear mechanism for selecting grouping variables and making between-group comparisons, since this is central to how researchers interpret ecological variation in microbiome studies. Clear communication about which metrics are exact and which are derived approximations is especially important for scientific transparency. [file:1]

## Example Use Case

A user analyzing rat gut microbiome data in MicrobiomeAnalyst can export the Observed Features and Shannon alpha-diversity result tables, upload both into MicroEven, choose treatment as the grouping variable and week as the faceting variable, and calculate Pielou’s evenness for each sample. The user can then generate a boxplot or violin plot, run a mixed model if subjects were followed over time, and export both the results table and figure for insertion into a manuscript draft. [file:1]

## Limitations

The current version does not ingest full taxonomic abundance tables and therefore cannot calculate true Simpson evenness unless a Simpson metric is separately provided in future development. The code explicitly labels its alternate bounded metric as Shannon-derived equitability to avoid overstating what can be inferred from Shannon and observed richness alone. [file:1]

The application is also focused on alpha-diversity post-processing rather than complete microbiome analysis. Users still need MicrobiomeAnalyst or another upstream workflow to generate the initial diversity outputs. [file:1]
